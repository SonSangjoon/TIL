## 객체지향 프로그래밍 OOP

- 추상화
  * 데이터 타입의 표현과 연산을 캡슐화
  * 접근 제어를 통해 데이터의 정보를 은닉
- 상속
  *  하위 클래스를 이용해 프로그램의 요구에 맞추어 클래스 수정
  * 클래스 간의종속 관계를 형성하여 객체를 조직화
- 다형성
  * 오버라이딩: 같은 이름의 메서드가 여러클래스에서 다른 기능을 하는것 / 상속으로 물려 받은 자료나 메서드를 하위클래스에서 새로 정의해 사용하는 기법
  * 오버로딩: 클래스 내부에 동일한 이름의 행위를 여러개 정의하는 것 /  매서드의 이름은 같고 매개변수의 타입과 수는 서로 달라야함

개체: 인스턴스(메모리에 로)

매서드: 객체가 가지고 있는 메서드를 호출한다.
- 한 객체의 속성을 조작하기 위해 사용
